# 企業資訊安全營運中心 (SOC) 標準作業程序與事件響應指南 v2.4

## 1. (Incident Severity & Escalation)
為了確保資安事件能被及時且適當地處理，所有異常通報均需依據以下等級進行歸類與響應：

- **Critical (嚴重)**: 
  - 觸發條件：核心資料庫 (如客戶個資、財務系統) 遭入侵、勒索病毒 (Ransomware) 橫向移動、或是公司主要對外服務遭受巨量 DDoS 攻擊導致中斷。
  - 處置時效 (SLA)：15 分鐘內響應。  
  - 通報層級：需立即通報 CISO (資安長) 與 IT 主管。必須強制執行網路實體隔離。
- **High (高)**: 
  - 觸發條件：偵測到外部惡意 IP 持續對內部網段進行 Port Scan、針對 VPN 或 SSH 服務的暴力破解嘗試、或是內部主機連線至已知的惡意中繼站 (C&C Server)。
  - 處置時效 (SLA)：1 小時內響應。
  - 通報層級：SOC L2 分析師。需對惡意來源 IP 進行防火牆封鎖審批。
- **Medium (中)**: 
  - 觸發條件：員工終端電腦 (Endpoint) 偵測到潛在惡意軟體 (如釣魚郵件附檔)，但已被 EDR/防毒軟體成功攔截並隔離。
  - 處置時效 (SLA)：8 小時內響應。
  - 通報層級：SOC L1 分析師。需追蹤該員工的後續登入行為。
- **Low (低)**: 
  - 觸發條件：一般廣告軟體 (Adware)、非關鍵系統的異常登入失敗 (低於 3 次)、或是防火牆外部的常規背景掃描 (Internet Noise)。

## 2. 威脅情資 (Threat Intelligence) 與 IP 處置規範

### 2.1 外部 IP 威脅分析標準
當系統偵測到未知的外部 IP 連線時，分析師必須綜合外部 API 情資與內部歷史資料庫進行交叉比對：
1. **地理位置 (Geo-IP) 異常**：若連線來源屬於公司業務未涵蓋之高風險國家，基礎風險分數需增加 30 分。
2. **雲端與匿名服務商**：若 IP 隸屬於匿名代理 (Tor Node) 或是常被駭客濫用的雲端主機商 (如部分廉價 VPS)，需標記為重點觀察對象。
3. **內部歷史情資**：必須查詢內部 SQLite 信譽資料庫。若該 IP 曾被標記為 `Botnet` (殭屍網路)、`APT` (進階持續性威脅) 或 `Scanner`，應直接判定為 High 等級威脅。

### 2.2 防火牆 IP 封鎖審批機制 (Human-in-the-Loop)
**【極度重要】** 為避免誤封鎖公司正常業務來源或重要合作夥伴，AI 系統或 L1 分析師**不具備自動封鎖 IP 的最終權限**。
1. 當系統判斷某 IP 必須被封鎖時，必須發出 **「授權審批 (Approval Request)」**。
2. 系統需列出該 IP 的：地理位置、內部信譽紀錄、觸發封鎖的原因。
3. 必須由具有權限的人類分析師點擊「授權封鎖」後，方可將該 IP 寫入內部防火牆黑名單。
4. 若分析師判斷為誤判 (False Positive)，可點擊「拒絕並觀察」，系統應將該 IP 列入灰名單持續監控 72 小時。
5. **重複封鎖防範**：若內部威脅情資資料庫顯示該 IP 已在黑名單中，代表防火牆已生效。分析師僅需回報其封鎖狀態與歷史原因，**嚴禁重複發起封鎖審批**。

## 3. 常見攻擊劇本標準作業程序 (Playbooks)

### 3.1 暴力破解攻擊 (Brute Force Attack) 處置
- **判斷標準**：單一 IP 在 5 分鐘內出現超過 10 次 `Failed password` 或 `Login failed` 日誌。
- **處置動作**：
  1. 透過 Threat Intel 工具查詢該 IP 是否為已知攻擊來源。
  2. 暫時限制該 IP 的連線頻率 (Rate Limiting)。
  3. 若該 IP 嘗試登入的帳號為高權限帳號 (如 root, admin)，觸發 High 等級事件，並提報防火牆封鎖。

### 3.2 殭屍網路與惡意探測 (Botnet & Malicious Probing)
- **判斷標準**：大量分散式 IP 嘗試尋找系統漏洞，或查詢內部資料庫確認為歷史 Botnet 節點。
- **處置動作**：
  1. 立即啟動 WAF (網頁應用程式防火牆) 的阻擋模式。
  2. 提取 Botnet 的特徵碼 (Signature)，更新至內部防護系統。
  3. 針對確認為惡意的 Botnet IP，嚴格執行封鎖審批流程。

### 3.3 網頁應用程式攻擊 (Web Attack)
- **判斷標準**：HTTP 請求的 URI 或 Payload 中出現 `UNION SELECT`, `OR 1=1` (SQL Injection 特徵)，或是 `<script>`, `alert()` (XSS 特徵)。
- **處置動作**：
  1. 阻擋該次惡意請求。
  2. 檢查資料庫是否有被異常撈取大筆資料的行為 (Data Exfiltration)。
  3. 將攻擊者的 IP 提報分析，若為惡意代理伺服器則提報封鎖。
